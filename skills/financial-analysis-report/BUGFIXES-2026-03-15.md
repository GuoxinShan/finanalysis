## Summary

3. **Wrong data extraction** - Extracts equity instead of profit for attributable profit

## Bug #1: File Overwrite (CRITICAL)

### The Problem

When parsing 2 PDFs for YoY comparison, **both wrote to the same file**:

```python
# BEFORE (BROKEN):
fs_index_2024 = parse_pdf(args.pdf_2024, args.company, args.output_dir)
# Returns: output/CHINHIN/fs_index.json

fs_index_2023 = parse_pdf(args.pdf_2023, args.company, args.output_dir)
# ALSO returns: output/CHINHIN/fs_index.json ❌ OVERWRITES 2024 DATA!
```

**Impact:**
- 2024 data completely lost
- Both `fs_index_2024` and `fs_index_2023` point to the same file (2023 data)
- All YoY growth calculations show 0% (comparing identical data)
- Users won't notice until they see all metrics are zero

### The Fix

Parse to **year-based subdirectories**:

```python
# AFTER (FIXED):
fs_index_2024 = parse_pdf(args.pdf_2024, args.company, args.output_dir)
# Returns: output/CHINHIN/2024/fs_index.json

fs_index_2023 = parse_pdf(args.pdf_2023, args.company, args.output_dir)
# Returns: output/CHINHIN/2023/fs_index.json ✓ SEPARATE FILES!
```

**Implementation:**
1. Parse to temporary directory first
2. Extract fiscal year from `fs_index.fiscal_year_end`
3. Create year-based subdirectory: `output/CHINHIN/2024/`
4. Copy `fs_index.json` to final location

## Bug #2: No Period Validation (HIGH)

### The Problem

The script accepted **any two PDFs** without validation:

```bash
# This would run without error (but make no sense):
python generate_report.py \
  --pdf-2024 CHINHIN_2024.pdf \
  --pdf-2023 CHINHIN_2024.pdf \  # Same year!
  --company CHINHIN
```

**Impact:**
- Same-year comparisons produce meaningless results
- Swapped arguments (2023 as current, 2024 as prior) cause confusion
- No error message to alert user

### The Fix

Added **period validation** with clear error messages:

```python
def validate_periods(fs_index_path: str, prior_fs_index_path: Optional[str]) -> None:
    """Validate that periods are different and in correct order"""
    current_year = current_fs.get('fiscal_year_end', '')[:4]
    prior_year = prior_fs.get('fiscal_year_end', '')[:4]

    # Check 1: Different years
    if current_year == prior_year:
        raise ValueError(
            f"❌ ERROR: Both PDFs are from the same year ({current_year}). "
            f"Current PDF and prior PDF must be from different periods.\n"
            f"   Current: {fs_index_path} (FY{current_year})\n"
            f"   Prior: {prior_fs_index_path} (FY{prior_year})"
        )

    # Check 2: Correct order
    if int(current_year) <= int(prior_year):
        raise ValueError(
            f"❌ ERROR: Current period (FY{current_year}) must be later than "
            f"prior period (FY{prior_year}). You may have swapped the arguments."
        )
```

**Error Examples:**

```
❌ ERROR: Both PDFs are from the same year (2024).
   Current: output/CHINHIN/2024/fs_index.json (FY2024)
   Prior: output/CHINHIN/2024/fs_index.json (FY2024)
```

```
❌ ERROR: Current period (FY2023) must be later than prior period (FY2024).
   You may have swapped the --pdf-2024 and --pdf-2023 arguments.
```

## Bug #3: Wrong Data Extraction (MODERATE)

### The Problem

The attributable profit extraction matched **any key containing "attributable to owners"**:

```python
# BEFORE (BUGGY):
for key in fs_index['line_items'].keys():
    if 'attributable to owners' in key.lower():  # Too broad!
        attr_current = fs_index['line_items'][key].get('group_current')
        break
```

**Impact:**
This could match:
- ✅ "profit for the financial year attributable to owners of the parent" (CORRECT)
- ❌ "equity attributable to owners of the parent" (WRONG - this is balance sheet equity!)

In sample reports:
- Attributable profit: RM809,973k
- PAT: RM215,492k
- **Attributable is 3.8x larger than PAT** - mathematically impossible!

### The Fix

Require **both "profit" AND "attributable to owners"**:

```python
# AFTER (FIXED):
for key in fs_index['line_items'].keys():
    key_lower = key.lower()
    if 'profit' in key_lower and 'attributable to owners' in key_lower:
        attr_current = fs_index['line_items'][key].get('group_current')
        break

# Sanity check: Attributable profit should be <= PAT
if attr_current and pat_current and attr_current > pat_current * 1.01:
    print(f"⚠️  WARNING: Attributable profit ({attr_current}) > PAT ({pat_current})")
    print(f"   This is unusual. Attributable profit should typically be <= PAT.")
```

## Additional Improvements

### Auto-detection for Cached Data

When using `--skip-pdf-parsing`, the script now:
1. Auto-detects year-based subdirectories: `output/CHINHIN/*/fs_index.json`
2. Sorts by year (newest first)
3. Validates periods even with cached data

```python
fs_index_files = glob.glob(os.path.join(args.output_dir, '*/fs_index.json'))
fs_index_files.sort(reverse=True)  # Newest first

fs_index_2024 = fs_index_files[0]  # Latest year
fs_index_2023 = fs_index_files[1] if len(fs_index_files) > 1 else None
```

### Period Detection from fs_index

Period is now determined from **actual fs_index metadata** instead of guessing from filename:

```python
period_year = extract_year_from_fs_index(fs_index_2024)
period = f"FY{period_year}"
```
