# Data Format Specification

Detailed specification of data formats used by the financial-analysis-report skill.

---

## Overview

The skill uses a **hybrid extraction approach**:
- **Structured financial data**: Extracted from `fs_index.json` (100% accurate)
- **Qualitative text data**: Searchable via `text_blocks.jsonl` (LLM-extracted)
- **Multi-year trends**: Calculated from multiple years of data

---

## Input Files

### fs_index.json

**Purpose**: Structured financial statement data parsed by finanalysis CLI

**Structure**:
```json
{
  "metadata": {
    "company_name": "CHINHIN",
    "fiscal_year_end": "2024-12-31",
    "currency": "MYR",
    "source_file": "annual_report_2024.pdf",
    "parsed_at": "2026-03-16T10:30:00Z"
  },
  "line_items": {
    "Revenue": {
      "group_current": 3252347,
      "group_prior": 2057210,
      "company_current": 3200000,
      "company_prior": 2000000,
      "unit": "RM'000"
    },
    "Profit Before Tax": {
      "group_current": 275845,
      "group_prior": 189318,
      "unit": "RM'000"
    },
    "Total Assets": {
      "group_current": 2156489,
      "group_prior": 1634271,
      "unit": "RM'000"
    }
    // ... 236 line items total
  }
}
```

**Key Fields**:
- `group_current`: Current year consolidated value (most common)
- `group_prior`: Prior year consolidated value
- `company_current`: Current year company-only value (excluding subsidiaries)
- `company_prior`: Prior year company-only value
- `unit`: Always "RM'000" (thousands of Malaysian Ringgit)

**Data Quality**:
- **Accuracy**: 100% (extracted from structured tables, not LLM)
- **Completeness**: ~236 line items covering balance sheet, income statement, cash flow
- **Verification**: Each value traced back to specific table cell in PDF

### metrics.json

**Purpose**: Pre-calculated financial ratios and derived metrics

**Generated by**: `finanalysis calculate` command

**Structure**:
```json
{
  "profitability": {
    "gross_margin": {"current": 16.15, "prior": 9.16},
    "operating_margin": {"current": 8.48, "prior": 9.19},
    "pat_margin": {"current": 6.63, "prior": 7.91},
    "roe": {"current": 23.4, "prior": 21.9},
    "roa": {"current": 10.0, "prior": 9.9}
  },
  "solvency": {
    "current_ratio": {"current": 1.32, "prior": 1.45},
    "quick_ratio": {"current": 0.89, "prior": 0.95},
    "debt_to_equity": {"current": 0.34, "prior": 0.20},
    "interest_coverage": {"current": 8.2, "prior": 12.5}
  },
  "growth": {
    "revenue_growth": {"current": 58.1, "prior": 13.0},
    "pat_growth": {"current": 32.4, "prior": -5.2},
    "asset_growth": {"current": 32.0, "prior": 15.0}
  },
  "cash_flow": {
    "ocf_to_pat": {"current": -0.28, "prior": 1.12},
    "fcf_margin": {"current": -1.8, "prior": 4.2}
  }
}
```

**Calculation Notes**:
- All percentages are decimals (16.15 means 16.15%)
- Calculated from raw RM'000 values before rounding
- See `references/precision_quick_reference.md` for rounding standards

---

### text_blocks.jsonl

**Purpose**: Extracted text from PDF for qualitative analysis

**Format**: JSONL (one JSON object per line)

**Structure** (each line):
```json
{
  "page": 42,
  "block_type": "text",
  "content": "Management Discussion and Analysis\n\nThe construction sector showed strong recovery...",
  "bbox": [50, 100, 550, 700],
  "confidence": 0.95
}
```

**Usage**:
- Workers search this file for qualitative information
- No fragile regex patterns - LLM extracts what it needs
- Page hints narrow search space

### Individual Worker Bundles (worker_N_bundle.json)

**Purpose**: Smaller, worker-specific data files for efficiency

**Generated by**: `scripts/extract_worker_bundle.py`

**Structure**: Same as `data_bundles.json` but only contains one worker's data

**Example**: `worker_2_bundle.json`
```json
{
  "worker_2": {
    "metadata": { ... },
    "metrics": { ... },
    "margins": { ... },
    "yoy_analysis": { ... }
  }
}
```

**Size**: ~200-500 lines vs. ~2000+ lines for full data_bundles.json

**Benefits**:
- Faster file reads
- Less context pollution
- Easier debugging

## Data Flow

```
PDF Report
    ↓ (finanalysis CLI)
fs_index.json (structured financial data)
    ↓ (finanalysis calculate)
metrics.json (calculated ratios)
    ↓ (data_extractor.py)
data_bundles.json (worker-specific bundles)
    ↓ (extract_worker_bundle.py) [OPTIONAL]
worker_N_bundle.json (individual bundles)
    ↓ (passed in worker prompts)
Worker Agents (generate report sections)
    ↓
worker_N_output.md (section content)
    ↓ (assemble_report.py)
Final Report (9 sections)
    ↓ (Worker 7)
Executive Summary (4 sections)
```

## Data Quality Standards

### Precision

**Rule**: Calculate from source data BEFORE rounding. Never use rounded values as inputs.

**Example**:
```
❌ WRONG:
   PAT: 215.5m, PATMI: 114.8m
   NCI = 215.5 - 114.8 = 100.7m  ← ERROR

✅ CORRECT:
   NCI = 215,492 - 114,818 = 97,078
   Display: 97.1m
```

See `references/precision_quick_reference.md` for details.

---

### Verification Metadata

Every data bundle includes `_verification` section:

```json
{
  "_verification": {
    "data_source": "fs_index.json",
    "extraction_method": "data_extractor.py v3.0",
    "data_quality": "REAL_DATA_EXTRACTED",  // or "PLACEHOLDER_DATA"
    "extracted_at": "2026-03-16T10:35:00Z",
    "line_items_count": 236,
    "metrics_calculated": true
  }
}
```

**Quality Indicators**:
- `data_quality: "REAL_DATA_EXTRACTED"` - Good, use this data
- `data_quality: "PLACEHOLDER_DATA"` - Bad, extraction failed
- `line_items_count: 236` - Should be > 100 for annual reports

## Common Data Issues

### Issue: data_bundles.json is empty or has zeros

**Diagnosis**:
```bash
# Check if fs_index.json has data
jq '.line_items | length' output/2024/fs_index.json
# Should be > 100

# Check verification metadata
jq '._verification' workspace/data_bundles.json
```

**Solution**: Re-run finanalysis CLI and data extraction

---

### Issue: Multi-year trends missing

**Diagnosis**:
```bash
jq '._multi_year_trends' workspace/data_bundles.json
# Should show years array
```

**Solution**: Ensure you provided multiple years with `--prior` flags:
```bash
python scripts/data_extractor.py output/2024/fs_index.json \
  --prior output/2023/fs_index.json \
  --prior output/2022/fs_index.json \
  --output workspace/data_bundles.json
```

---

### Issue: Inconsistent metrics across workers

**Root Cause**: Workers reading different data sources or recalculating

**Solution**:
1. Use same `data_bundles.json` for all workers
2. Pass data directly in prompts (workers don't read files)
3. Verify all workers receive same base data

## File Size Guidelines

| File | Typical Size | Lines | Notes |
|------|-------------|-------|-------|
| fs_index.json | ~50-100KB | ~2000 | Structured financial data |
| metrics.json | ~5-10KB | ~200 | Calculated ratios |
| text_blocks.jsonl | ~500KB-2MB | ~5000-10000 | Extracted text (optional) |
| data_bundles.json | ~100-200KB | ~2000-3000 | All worker bundles |
| worker_N_bundle.json | ~10-30KB | ~200-500 | Individual worker bundle |

## Best Practices

1. **Always verify data quality** before spawning workers:
```bash
jq '._verification.data_quality' workspace/data_bundles.json
# Should be "REAL_DATA_EXTRACTED"
```

2. **Use individual worker bundles** for efficiency:
```bash
python scripts/extract_worker_bundle.py --all \
  --input workspace/data_bundles.json \
  --output-dir workspace/bundles
```

3. **Check multi-year trends** if using 3+ years:
```bash
jq '._multi_year_trends.years' workspace/data_bundles.json
# Should be ["2024", "2023", "2022"]
```

4. **Validate before assembly**:
```bash
# Check all worker outputs exist
ls -l workspace/worker_*_output.md

# Verify Worker 6 has 3 sections
grep -c "^# [ⅥⅧⅨ]" workspace/worker_6_output.md
# Should be 3
```

## Summary

**Input Data**:
- `fs_index.json`: Structured financial data (100% accurate)
- `metrics.json`: Pre-calculated ratios
- `text_blocks.jsonl`: Qualitative text (optional)

**Intermediate Data**:
- `data_bundles.json`: All worker bundles (~2000-3000 lines)
- `worker_N_bundle.json`: Individual bundles (~200-500 lines)

**Output Data**:
- `worker_N_output.md`: Section content from workers
- `*-revised.md`: Final 9-section report
- `*-summary.md`: Executive summary

**Data Flow**: PDF → fs_index.json → data_bundles.json → workers → final report
